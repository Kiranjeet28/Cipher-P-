Figure 5 — Hill Cipher (Matrix-Based Block Cipher)
n×n matrix multiplication mod 256 on blocks of n RGB bytes. Supports n=2 and n=3.

Figure 5: Hill cipher flowchart. Key validation (determinant coprime to 256) occurs before any encryption. Matrix inverse computed once; reused for all blocks.

Figure 6 — Playfair Cipher (16×16 Extended Digraph Substitution)

Figure 6: Playfair 16×16 cipher. The three positional rules are applied per byte pair. Rectangle rule is its own inverse; row/column rules shift direction reverses on decrypt.

Figure 7 — RC4 Stream Cipher (KSA + PRGA)

Figure 7: RC4 KSA (left) initialises the 256-byte S-box from the key. PRGA (right) generates one keystream byte per input byte via S-box state updates; XOR produces ciphertext. Encrypt = Decrypt.

Figure 8 — Rail Fence Cipher (Transposition)

Figure 8: Rail Fence encrypt (left) writes bytes to rails in zigzag order, then reads rails sequentially. Decrypt (right) reverses by precomputing rail assignments and remapping positions.

How the backend routes algorithm string identifiers to concrete cipher objects

Figure 9: CipherFactory::createCipher() dispatch logic. Each algorithm string maps to a concrete cipher class. All implement the CipherBase interface (encrypt, decrypt, validateKey).
